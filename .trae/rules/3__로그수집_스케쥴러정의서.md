# PreCheck 로그 수집 스케쥴정의서 v1.0

## 문서의 목적
- 로그 수집 시스템에서 참고할 수집 대상서버, 대상서버내 파일 위치/수집 주기를 정의한 파일을 설명
- 로그 수집 시스템내 파일 형태로 정의함

---

## 로그 수집 정의서 파일명
- 로그 수집 시스템 내 ~/cfg/PreCheck_CollectLogs_Schedule.conf

## 로그 수집 주기
- 로그 수집 주기는 다음과 같음
- 배치 -> 일회성 로그 수집, 로그 전체
- 주기 -> 정해진 시간부터 주기별로 로그 수집, 최초 수집은 로그 파일 전체, 이후 주기부터는 추가된 로그만을 수집(마지막으로 읽은 라인번호로 판별)

## 로그 수집 정의 파일 포맷
- 로그 수집 정의 파일의 포맷은 다음과 같음
- [서버구분][아이피주소][대상파일명(절대경로)][수집주기기술]
    -> 서버구분 : 대상서버의 호스트명 또는 서버사용명(ex 서버자동주문)
    -> 아이피주소 : 대상서버의 아이피주소
    -> 대상파일명 : 대상서버내 로그 위치, 절대경로
    -> 수집주기기술 : 수집구분, 수집시간, 수집주기

- 수집주기기술은 수집구분별로 다음의 포맷으로 정의
    -> 배치 : [배치|요일|시작시간]
    -> 주기 : [주기|요일|시작시간|수집간격(분)|종료시간]
        -> 요일 표기 규칙
            * 단일 요일 : 0~6 중 하나 (0->일요일, 1->월요일, 2->화요일, ..., 6->토요일)
            * 범위      : 시작-끝 형식 (예: 1-5 = 월~금)
                          - 시작은 끝보다 작거나 같아야 함
                          - 0-6은 *과 동일 (매일)
            * 전체      : * (매일)
            * 콤마(,) 또는 복합 표기(예: 1,3,5 또는 1-3,5)는 1차 버전에서 미지원
            * 위 규칙을 따르지 않는 요일 표기는 스케쥴 무시 처리
        -> 시작시간 : HHmmss, 시작시간은 000001이 최소임
        -> 수집간격(분) : 분단위(예 5:5분)
        -> 종료시간 : HHmmss, 종료시간은 235959가 최대임

- 파일은 텍스트 형태며 개행문자(\n)로 구분
- 위 포맷을 따르지 않는 스케쥴은 무시함
- skip처리는 #으로 시작함
- 수집 실패 시 재시도 정책은 프로그램 명세서를 따름(최초 시도 실패 후 10초 간격 재시도 3회, 배치/주기 공통 적용)

## 스케쥴 예
- #[dlprem01-테스트개발][192.168.210.121][/tmp/check.out][배치|3|090001] -> skip
- [dlprem01-테스트개발][192.168.210.121][/tmp/check.out][배치|3|090001] -> 수요일 9시 00분 01초에 한번 수집
- [dlprem01-테스트개발][192.168.210.121][/tmp/check.out][배치|1-6|090001] -> 월~토요일 9시 00분 01초에 한번 수집
- [dlprem01-테스트개발][192.168.210.121][/tmp/check.out][배치|*|090001] -> 매일 9시 00분 01초에 한번 수집
- [dlprem01-테스트개발][192.168.210.121][/tmp/check.out][주기|*|090001|10|100001] -> 매일 9시 00분 01초부터 10분간격으로 10시 00분 01초까지 수집
- [dlprem01-테스트개발][192.168.210.121][/tmp/check.out][주기|1-5|090001|10|100001] -> 월~금요일 9시 00분 01초부터 10분간격으로 10시 00분 01초까지 수집
